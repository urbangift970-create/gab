What is the Devnet, Testnet and Mainnet?

1. Devnet
(Development Network)

Devnet is primarily used by developers for testing and experimentation. It’s an early-stage network where new features, updates, or changes to a blockchain project can be tried out in a controlled environment.

The Devnet is often less stable and might have frequent changes. It’s not intended for long-term use or production-ready applications. It is typically accessible only to developers and partners involved in the project as it’s a space for testing and development before moving to more public testing.

2. Testnet
(Testing Network)

Testnet is used for broader testing of blockchain protocols and applications, allowing developers and users to experiment with the network without risking real assets. It simulates the mainnet environment but uses test tokens or assets that hold no real-world value.

Testnets are more stable than devnets and are used to detect and fix issues before deploying changes to the mainnet. It helps in validating performance and security under more realistic conditions.

3. Mainnet
(Main Network)

Mainnet is the live, fully operational blockchain network where real transactions occur, and real assets are transacted. It represents the final, production version of the blockchain.

Mainnet is designed to be stable, secure, and capable of handling real-world usage. Changes and updates are carefully controlled and rolled out to maintain network integrity.

Summary

Devnet: Experimental, developer-focused network for initial testing and development.

Testnet: Publicly accessible network for broader testing and simulation before changes are applied to the mainnet.

Mainnet: The live, production network where real transactions and interactions occur.

Why is a Testnet important?

Risk-Free Testing

Developers can experiment with new code, smart contracts, or decentralized applications (dApps) without the risk of financial loss. Since testnet tokens have no real value, testing can be done freely.

Identifying Bugs and Issues

Testnets allow developers to find and fix bugs, vulnerabilities, or inefficiencies in their code before it affects real users. This helps ensure the security and stability of the mainnet once features are deployed.

Consensus Mechanism Testing

Changes to a blockchain’s consensus mechanism (like Proof of Stake or Proof of Work) can be tested on a testnet to verify their performance and impact on the network without disrupting the live chain.

Scalability and Performance Testing

Testnets enable performance testing under different loads and conditions, helping developers assess scalability, transaction speeds, and potential bottlenecks before launching on the mainnet.

Mining & Node Testing

Miners and node operators can use testnets to configure their hardware and software, ensuring compatibility with new network updates, smart contracts, or governance changes.

Community and Ecosystem Testing

Communities and projects can use testnets to educate new users and onboard developers by providing a sandbox environment for experimentation.

Smart Contract Development

For smart contract developers, testnets are where they can deploy, interact with, and test their contracts with a real blockchain environment but with fake assets. This ensures functionality and security.

Summary

In summary, blockchain testnets are a safe, controlled environment for conducting experiments, ensuring stability, and improving the reliability of blockchain features before they go live on a mainnet.

The BlockDAG Testnet provides all the core functionality that is required to launch and operate a Layer 1 blockchain, including:

BlockChain Nodes

Multiple nodes running the BlockDAG software all collaborate and contribute to the functionality and security of the blockchain network:

Miners

Wallet Connection

Allows connection to MetaMask wallet

Explorer

A web-based tool that allows users to interact with and explore data on the BlockDAG blockchain in a transparent and user-friendly way. It provides a range of valuable functions and insights about the blockchain's activities, including transaction history, block details, smart contract interactions, and more.

Faucet

This is a tool normally included in the Explorer that is unique to the Testnet. It allows users to mint coins and transfer them to their wallet address.

I’m ready to try it out – what do I do now?

In order to get started we suggest you follow these steps:

Getting Started

  • Create your BDAG wallet address:
    • Install MetaMask (if it is not already installed)
    • Connect it to the BlockDAG Testnet
    • Create a BDAG Wallet address
  • Create your BDAG wallet address:
    • Open the BlockDAG Explorer in your browser
    • Navigate to the Faucet menu item
    • Once in the faucet, select the number of BDAG coins (you can mint 100 BDAG coins per day) that you wish to mint, enter your wallet address, and go. This will call a faucet smart contract that will mint the coins in your wallet address.
  • ⁠Check MetaMask to see that the coins have been received. Your balance should be 100 coins x 0.05c = $5.00 (this may take up to 5 minutes).

Send and Receive BDAG

  • Send to a second address, or to a friend
  • Receive BDAG from a friend, or another address you have setup.
