Blockchain update

We are pleased to share some exciting progress on our blockchain explorer! Our development team has completed the faucet page, and we are now in the process of deploying the infrastructure on the Devnet.

Devnet Deployment
We are currently in the phase of deploying our infrastructure on the Devnet. This deployment is essential for testing the stability and performance of our blockchain explorer before we move to the mainnet. Here are some key components of our infrastructure:

  1. Node Setup: Our infrastructure includes multiple nodes that communicate with each other to maintain the decentralized network. These nodes are configured to ensure high availability and reliability.
  2. Load Balancing: To handle multiple requests simultaneously and ensure smooth performance, we have implemented load balancers. This will distribute the traffic evenly across our nodes.
  3. Database Integration: A robust database system is in place to store blockchain data securely. This includes transaction histories, block details, and user data for the faucet functionality.
  4. API Gateway: We have developed an API gateway that allows seamless interaction with our blockchain explorer. This gateway is designed to handle various requests efficiently, providing users with real-time data and updates.
  5. Security Measures: Security is our top priority. Our infrastructure includes advanced security protocols to protect against potential threats and ensure data integrity and privacy.
     

AWS Server Setup Configurations
To ensure optimal performance and reliability of our blockchain explorer, we have configured our AWS server with the following settings:

  1. EC2 Instances: We are using Amazon EC2 instances to host our nodes and other services. These instances are configured for high availability and auto-scaling to handle varying loads.
  2. Elastic Load Balancing (ELB): ELB is set up to distribute incoming traffic across multiple EC2 instances, ensuring no single instance is overwhelmed. This helps maintain high availability and fault tolerance.
  3. RDS (Relational Database Service): We have integrated Amazon RDS for our database needs. RDS provides automated backups, software patching, and seamless scaling of storage and compute resources.
  4. Amazon S3: For storage of blockchain data and logs, we are using Amazon S3. S3 provides durable and scalable storage solutions, ensuring our data is safe and easily accessible.
  5. VPC (Virtual Private Cloud): Our infrastructure is deployed within a VPC to isolate our resources and control network access. This enhances security and provides a customizable network environment.
  6. IAM (Identity and Access Management): IAM policies are in place to control access to AWS resources. We have defined roles and permissions to ensure only authorized personnel can access critical components of our infrastructure.
  7. CloudWatch: We are using Amazon CloudWatch for monitoring our infrastructure. CloudWatch provides real-time insights and alerts, helping us maintain the health and performance of our services.
     

Next Steps: Binding the URL
Once the Devnet deployment is complete, our next step will be to bind the URL to our blockchain explorer. This will make the faucet page and other features accessible to users. We are committed to ensuring that this process is swift and seamless, providing you with a reliable platform to interact with our blockchain.
Stay tuned for more update!
