BlockDAG Ignites the Future: BEVM Upgrade, X-Series Miner Rollout

This update represents a major stride in the blockDAG ecosystem, especially as we transition to a more stable and Ethereum-compatible infrastructure. 
Our current focus has been on:

  • Upgrading the blockDAG EVM (BEVM) to match the latest Ethereum stack
  • Refining explorer frontend to support advanced search and query flows
  • Strengthening the mobile app (X1) for miner/device security
  • Preparing the firmware stack for X10 and initiating field-level tests on the X30 miner
  • Fixing critical UI/UX bugs and backend inconsistencies reported by the community


This release improves testnet readiness, hardware mining stability, and user accessibility through frontend/UI changes.

Blockchain Updates

Protocol Core: BEVM Upgrade to Go-Ethereum v1.14.11


Objective: To make the blockDAG EVM (BEVM) fully compatible with Ethereum tooling while preserving DAG-optimized consensus mechanisms.


Migration Process

  • Rebased the BEVM on top of Go-Ethereum v1.14.11, inheriting:
    • JSON-RPC interface upgrades
    • Sync protocol enhancements (snap sync improvements)
    • EVM opcode compliance updates (relevant to EOF previews)
  • Applied internal patches to maintain blockDAG-specific graph data structures (non-linear parent references)
  • Updated block finalization logic to comply with higher-order DAG reconciliation and weighted voting consensus


Current Testing (Stage) & Next steps:

Testing is currently underway in multiple areas. Transaction throughput is being benchmarked with stress tests simulating 1,000 to 3,000 transactions per second. Node synchronization is being validated through single-node bootstraps from genesis using DAG-aware caching mechanisms. Graph consistency is under active verification via breadth-first traversal algorithms to ensure integrity in block referencing. On the EVM side, compatibility testing has been conducted through deployment and execution of ERC-20, ERC-721, and ERC-1155 contracts, including event emission validation.

Looking ahead, the team is working on integrating OpenTelemetry metrics to enable fine-grained instrumentation of the consensus process. Additional efforts include implementing snapshotting and optimizing state pruning, alongside preparing differential fuzz testing to validate edge-case behavior in DAG relationships.

X30 Miner – Hardware Compatibility Testing

The X30 miner is currently undergoing extensive hardware compatibility testing. During field evaluations, the device was successfully connected to a staging BlockDAG node using RPC communication, supplemented by a heartbeat module to maintain persistent synchronization. Mining difficulty was dynamically calibrated to reflect realistic network hash rate conditions, allowing for a more accurate simulation of live mining environments.
 

Initial observations have been promising. The device maintained a peak power draw of 2.1kW, which remains within operational tolerances. CPU utilization exceeded 88% during parallel DAG construction, indicating efficient use of compute resources under load. Additionally, nonce discovery rate across identical blocks exhibited only a 4.2% variance, a level of noise deemed acceptable for performance consistency.
 

Looking ahead, the team plans to conduct a sustained 48-hour stress test to assess thermal resilience and validate consistent reward integrity. Firmware updates for full BlockDAG compatibility will be delivered via over-the-air (OTA) deployment. Parallel efforts are also underway to develop a dashboard for aggregating live performance metrics from deployed X30 miners, enabling enhanced monitoring and diagnostics in real time.

BlockDAG Explorer Improvements (Frontend/UI)

Enhanced UI Features under progress and testing


1. Global Search Navigation

  • Implemented search auto-suggestions with:
    • Fuzzy matching of partial txn hashes, addresses, block IDs
    • Immediate linking to detail views
  • Added Next/Previous buttons to allow stepwise traversal of search results across paginated datasets


2. Relative Time Filters

  • Advanced filter section now interprets time as relative ranges:
    • “Last 1 Hour”, “Last 24 Hours”, “Last 7 Days”, etc.
  • Behind the scenes, this converts to Unix timestamps, improving query performance on backend time-indexed databases


3. Redirection Logic

  • Bug fix: Previously, advanced search navigation from a transaction detail page would reset state
  • Now, URL query parameters are persisted and restored using URLSearchParams + useRouter().push() logic

X1 Mobile App Integration with x10 Hardware Updates

X10 Device: The X10 hardware team is actively preparing for upcoming firmware upgrades to align with the recent changes in the architecture as per the new requirements and also in the design.
 

Meanwhile, the X1 mobile application team is addressing several vulnerabilities identified during a recent security audit of stage environment. Remediation efforts are underway to enhance the overall security posture of the app, ensuring safer device interactions and user authentication processes ahead of the next planned release.

Known Issues, Work in Progress & Next Milestones

The team continues to address several key areas under active development. The X1 mobile application is currently undergoing remediation for vulnerabilities identified during a recent security audit on the stage environment; these are being patched and retested to ensure user safety and application integrity. On the protocol side, post-upgrade tuning of the BlockDAG is in progress, with ongoing load testing designed to identify and resolve any performance bottlenecks. The X30 mining hardware is also being closely monitored, with continued optimization efforts underway to stabilize performance under realistic network loads.
 

Looking ahead, several major milestones are on the horizon. A patched and security-audited version of the X1 app is set to be released soon. Final testing and firmware synchronization for the X10 mining devices are progressing as scheduled. For the X30 units, mining stabilization efforts will be paired with the rollout of enhanced dashboard features for real-time metric tracking. On the backend, performance upgrades will begin surfacing within the Explorer analytics, offering the community deeper insight into system performance. Additionally, the launch of a new community dashboard is in the pipeline, aimed at providing transparent, live data around miner activity and network health.
