Elevating NFT Clarity, ERC Standard Accuracy & Runtime Planning in BlockDAG

This release brings meaningful improvements across the BlockDAG ecosystem from the way NFT transactions are displayed in the Explorer to fixes that make the mobile mining experience more reliable. We’ve rolled out a new API for NFT minting insights, improved how smart contracts are labeled, deployed a key blockchain contract for future wallet upgrades, and fixed several important user experience issues in the X1 mobile app.

At the same time, we’ve started research on how to upgrade the blockchain runtime safely and efficiently, ensuring that as BlockDAG grows, it remains stable, secure, and future-ready.

Purpose

To update the community on the latest Explorer, blockchain, and mobile app improvements. This release focuses on giving developers better tools, helping users better understand their transactions, and preparing the network for smooth, long-term upgrades.

Key Highlights

  • New API added for NFT minting transactions, showing token ID, metadata, minter address, and more.
  • Fixed wrong ERC-20 tags on contracts like TokenVesting that aren’t actual ERC-20 tokens.
  • Built and launched the new NFT module UI, following the finalized design.
  • Fixed how ERC-721 tokens show up in transaction details to display only relevant token IDs.
  • Deployed ERC-4337 EntryPoint contract on the Stage network to support smart wallet upgrades.
  • Started research on safe blockchain runtime upgrades.
  • Fixed syncing and UI issues in the X1 mobile app related to mining status and device interaction.

Explorer Updates

NFT Minting API

We’ve added a new API endpoint to make NFT minting events easier to track. Now, developers and users can get detailed info like

  • Token ID
  • Contract address
  • Minter’s wallet address
  • Transaction hash
  • Timestamp
  • Metadata link (IPFS/URL)

This is especially helpful for NFT marketplaces, wallets, and platforms that want to display minting history clearly.

 

Fixed ERC-20 Mislabeling

Some contracts were incorrectly labeled as ERC-20 even though they weren’t. We've fixed this by updating our contract detection logic to rely on the actual function signatures, not just metadata or names.
 

Improved ERC-721 Token Display

The /getTransactionDetails API is used to list all tokens linked to a contract, even if only a few were involved in a transaction. We’ve fixed this to show only the token IDs that were part of the actual transfer.

 

NFT Module Implementation

We’ve finished integrating the new design for the NFT module. This includes updated UI components, user flows, and backend integration, making NFT browsing and interaction much smoother and more user-friendly.

Blockchain Core Updates

ERC-4337 EntryPoint Deployment

We’ve deployed the ERC-4337 EntryPoint contract on the BlockDAG Stage network. This sets the foundation for account abstraction, allowing advanced wallet features like bundled transactions and sponsor-paid gas fees (via UserOperations).
 

Runtime Upgrade Research

We’ve started researching ways to upgrade the blockchain without causing network disruption. We’re exploring options like:

  • Swapping runtime components without restarting the chain
  • Allowing token holders to vote on upgrades
  • Keeping smart contract data safe and consistent during version changes
  • Rolling back upgrades safely if something goes wrong

This research will help us design a future-proof upgrade path for the BlockDAG chain.

X1 Mobile App Integration with x10 Hardware Updates

Mining Status Not Updating After Reset

When users reset their X10 mining device, it enters reset mode, but the app still shows it as “Online”. We’ve identified this issue and are working on a fix so the app reflects the correct status.
 

“Connect Device” Button Not Working

On the Archived Devices page, clicking “Connect Device” did nothing. This made it hard for users to reconnect to their mining hardware. This bug is being fixed for smoother device management.
 

Real-Time Mining Stats Not Updating

Mining time and token balances weren’t updating live in the X10 device list. Users had to refresh manually. We’re fixing this so it updates in real time while mining is in progress.

Open Issues

  • ‘To Address’ Balance Not Showing: When clicking the “To Address” in transaction details, the balance isn’t displayed, unlike the “From Address.” A fix is on the way.
  • Incomplete Event Logs: Some event logs on the transaction page don’t show full data like event names or indexed topics. We’re improving this to give developers better visibility.

What’s Next

  • Display token images, names, and descriptions in the NFT transaction and contract views.
  • Publish our blockchain upgrade plan after the research phase.
  • Improve event logs and function name rendering using advanced ABI parsing.
  • Upgrade the Node Visualizer with better controls and filters.

Final Thoughts

This update provides enhanced NFT visibility, improved contract detection, and key steps toward long-term blockchain enhancements. Whether you’re building on BlockDAG, mining with X1, or just exploring the network, we want your experience to be clear, smooth, and reliable.
 

Thanks for building the future with us, block by block.
