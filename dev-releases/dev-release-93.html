Dev Release 93

Release Date: August 9, 2024

  • Blockchain Explorer Update

Greetings BlockDAG Community,

Weekly Update: Progress and New Developments
This week, we made significant strides across various fronts, aligned with our roadmap. Here's a brief overview of the key activities:

  1. Login & Sign-Up System and Dynamic FAQs: We started the week by focusing on enhancing the login and sign-up processes, along with the implementation of dynamic FAQs to improve user experience.
  2. Notification Module: Parallel to the above tasks, our team began developing the notification management system, including the creation of an admin panel for easier notification control.
  3. Password Functionality Design: Midweek, we initiated the Figma designs for the new password functionality. Following the design completion, we received approval from our external stakeholders, allowing us to proceed.
  4. Logout Session Alert: To address user confusion regarding session termination upon logout, we're introducing an alert message that will prompt users with: "By logging out, the session will end. Do you want to continue?" This aims to clarify the logout process and prevent any misunderstandings.
  5. Phone Number API Improvement: We made enhancements to the phone number API to ensure more accurate data retrieval.
  6. Mining Rate Adjustments: One of the major discussions this week centered around mining rate adjustments. We've decided to keep the normal mining rate steady at 0.83 BDAG/hour. However, the referral mining rate will now be adjusted to 0.05 BDAG/hour. This adjustment reflects our transition from Devnet to Testnet and marks the successful near-completion of our Devnet phase. The referral rate change also signals the conclusion of our initial promotional phase, as we've surpassed the milestone of 100,000 users. This shift aligns with our original plan to reward early adopters and ensure sustainable growth moving forward.

Today’s Update:
 

  • Latest Build Submission: Last night, we successfully submitted the latest build of the BlockDAG X1 application to all major app stores, including:
  • Apple App Store
  • Huawei AppGallery
  • Amazon Fire Store

Upon submission, we received feedback from Apple’s review team concerning specific aspects of our data privacy policy.

Apple’s queries focused on how we handle, store, and protect user data within the application, reflecting their stringent privacy standards. To address these concerns, our team quickly prepared a detailed response: We outlined our data privacy measures, ensuring transparency in how we safeguard user information. The response was submitted back to Apple, and we are currently awaiting their review process to conclude. This review is critical for ensuring compliance with Apple’s regulations and maintaining the app’s availability on their platform.

 

  • Password Management Features:

Our development team has successfully implemented the "Change Password" functionality in the BlockDAG X1 app:

Users can now securely update their passwords directly from the app, enhancing overall account security.

This feature is part of a broader initiative to strengthen user authentication processes and reduce security vulnerabilities.

The implementation also includes user-friendly prompts and error messages to guide users through the password change process smoothly.

 

  • Deep Linking Implementation:

We received a new domain that will be used to implement deep linking within the app:

Deep linking allows users to be directed to specific content or pages within the app via external links, such as from emails, social media, or web pages.

This functionality will enhance user navigation by providing direct access to relevant app content, improving user engagement and satisfaction.

The deep linking feature has been in our backlog for some time, and with the new domain in place, we are ready to begin implementation.

The feature will be particularly useful for marketing campaigns, user onboarding, and content sharing, enabling a more integrated user experience.

 

  • Design Refinements: As part of our ongoing commitment to improving the user interface (UI), the team is working on several design refinements:
  • Color Coordination: We are enhancing the color scheme across the app to ensure a more visually appealing and cohesive look.
  • Text Placement: Adjustments are being made to the placement of text on various screens, ensuring better readability and easier interaction for users.
  • These design updates are driven by user feedback and aim to create a more intuitive and visually pleasing experience within the app.

Blockchain Explorer Update

UTXO Figma Design Adjustments:

  • The blockchain explorer team focused on refining the design for the UTXO (Unspent Transaction Output) model in Figma:
  • UTXO differs significantly from the EVM (Ethereum Virtual Machine) model, particularly in how transaction data is structured and displayed.
  • The design adjustments are necessary to accurately represent UTXO data while maintaining a user-friendly interface.
  • The blocks and transaction pages required specific adaptations to reflect the unique characteristics of the UTXO model, such as how outputs are tracked and spent.
  • The design team has been working closely with developers to ensure that the visual presentation aligns with the technical requirements of UTXO data.
     
  • Backend API Development:
  • Simultaneously, our backend team is working on developing APIs to support the blockchain explorer:
  • These APIs are essential for fetching, processing, and displaying blockchain data in real-time, ensuring users have access to the most current information.
  • The API development is being done with scalability in mind, allowing the explorer to handle large volumes of data and traffic efficiently.
  • The APIs will integrate with both the UTXO and EVM models, providing a comprehensive view of blockchain activity across different systems.
     
  • Next Steps:
  • Design Phase Completion: We anticipate completing the design phase by next week, after which we will move on to:
  • Front-End Integration: The development team will start integrating the finalized designs into the explorer’s front-end, ensuring that the visual elements are properly aligned with the backend functionality.
     
  • Rigorous Testing:
  • Data Consistency Testing: We will conduct thorough testing to ensure that data from both UTXO and EVM models is accurately represented and displayed within the explorer.
  • Load Testing: The explorer will undergo load testing to verify its ability to handle high volumes of traffic and large datasets without performance degradation.
  • User Interface (UI) Testing: Additional testing will focus on the user interface, ensuring that the explorer is intuitive, responsive, and free of bugs.
  • Deployment Readiness: Once testing is complete, we will prepare for the explorer’s deployment, ensuring that all components work seamlessly together.
  • Overall Goal:
  • Our primary objective is to deliver a robust, reliable blockchain explorer that provides users with comprehensive, accurate data access and high performance.
  • The explorer will serve as a critical tool for users to monitor and interact with the BlockDAG blockchain, supporting transparency and user engagement as we continue to scale.

Stay tuned week for more updates!
