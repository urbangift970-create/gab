Enhancing NFT Support, Strengthening Governance & Powering Launch Readiness in BlockDAG

This release introduces powerful enhancements across the BlockDAG ecosystem from expanded NFT media support and Trust Wallet integration in the Explorer to our pioneering governance contract for blockchain upgrades. With new export capabilities for transaction histories, strengthened vulnerability handling, and ongoing mobile mining fixes, we’ll be reinforcing transparency, usability, and long-term protocol governance.
 

Behind the scenes, we’re conducting planning meetings for the Awakening Explorer launch, supported by a comprehensive rollout playbook that ensures everything from feature validation to deployment strategy is covered.

Purpose

To share the latest updates in NFT visualization, multi-wallet access, governance mechanisms, and critical Explorer features, while preparing the ecosystem for scalable launches and secure blockchain upgrades.

Key Highlights

  • Add support for video and animated GIFs in the NFT module
  • Integrate Trust Wallet support into the BlockDAG Explorer
  • Enable CSV export on the Address and Contract Details pages
  • Introduce a governance smart contract for decentralized upgrade voting
  • Prepare the Awakening Explorer Launch Playbook for smooth deployment
  • Fix mining time misreporting in the X10 device list

Explorer Updates

Video and GIF Support for NFTs

NFTs aren’t just images; they’re immersive, multimedia experiences. We’ll be updating our metadata parsing logic to detect and display .mp4, .webm, and .gif files within the NFT view. Whether it's looping animations or short-form videos, users would be able to enjoy rich content directly within the Explorer.
 

Trust Wallet Integration

We add Trust Wallet support alongside MetaMask for seamless wallet connectivity on the Explorer. Users now interact with the BlockDAG network using either wallet, offering more flexibility and convenience across devices and platforms.
 

CSV Export for Addresses and Contracts

To make data analysis easier for developers and power users, we’ll be introducing CSV export functionality to both the Address Details and Contract Details screens. Users can export visible or complete transaction histories with a single click for external reporting or custom analytics.
 

Awakening Explorer Launch Playbook

We’re finalizing a robust playbook to guide the Awakening Explorer rollout. This document aligns every team for launch and includes:

  • Final feature checklist & versioning confirmation
  • Deployment and rollback strategy
  • Monitoring and alerting setup
  • Internal and public communication plans

Blockchain Core Updates

Governance Contract for Upgrade Voting

BlockDAG’s move toward decentralized governance takes a big leap with the development of a voting-enabled smart contract. This contract empowers miners and network participants to vote on blockchain upgrade proposals.

Key functionality includes:

  • Proposal creation and vote submission
  • Voting via signed addresses or on-chain actions
  • Enforcement of voting thresholds (e.g., majority or supermajority)
  • Vote validation and results finalization
  • Integration with runtime upgrade mechanisms (auto/manual)

We’re exploring models that ensure fairness and decentralization, whether voting is based on hash power or weighted address participation, while safeguarding the network against centralized control.
 

Vulnerability Resolution

We’re currently investigating a vulnerability issue. Our security and dev teams analyze the issue, assess impact, and determine mitigation strategies. Necessary fixes roll out promptly to ensure the network’s continued resilience and trustworthiness.

X1 Mobile App Fixes

Incorrect Mining Time Reset

We’ve identified an issue where the X10 mining timer resets to “24 hours” every time mining is stopped and resumed, regardless of the actual duration mined. This inflates perceived mining duration and misleads users. A fix is in progress to correctly accumulate mining time across sessions.

Closed Issues

‘To Address’ Balance Now Displays Correctly

We’ve resolved the issue where the “To Address” in transaction details didn’t show the balance, unlike the “From Address.” Both views now display balance data consistently, improving clarity for users tracking fund flows.
 

Improved Event Log Visibility

Event logs on transaction pages now include full data, such as event names and indexed topics. With enhanced ABI parsing in place, developers access more comprehensive smart contract insights directly from the Explorer.

What’s Next

  • Enable display of NFT token images, metadata, and names directly in transactions
  • Publish the finalized blockchain upgrade governance flow
  • Refine event logs and function name parsing across the Explorer
