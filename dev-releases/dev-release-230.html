Real-Time Explorer Sync, Relay Node Reliability Upgrade & X1/X10 Feature Planning Across the BlockDAG Ecosystem!

This update brings critical improvements to network reliability, user-facing explorer synchronization, and the next wave of miner-app functionality. The Explorer is moving toward live, stream-based data delivery, eliminating manual refresh requirements. On the blockchain side, relay node selection logic is being rebuilt with strict sync-state validation to prevent propagation delays or stale peer routing. Meanwhile, conceptual design and feature definition continue for the upcoming X1 and X10 app enhancements, ensuring smoother UX, clearer reward visibility, and improved device-level logic alignment across miner classes.

Key Highlights

Explorer

Live WebSocket pipeline under development for real-time block, transaction, and contract updates, removing dependency on manual page refreshes.
 

Blockchain

Relay node peer-selection logic upgrade: relay will connect only to fully synchronized peers (DAG + EVM), with strict fallback logic and automatic exclusion of partially synced nodes.
 

X1/X10 Miner

UI/feature planning for new in-app modules is underway, which includes new interactive features and interaction flows designed for the upcoming release of X1.

Explorer Updates

Transition to Real-Time Data Feeds

The current Explorer requires users to manually refresh the page to view new blocks, transactions, or contract deployments. To address this limitation, the development team is integrating a persistent WebSocket layer across all listing modules.
 

Core Improvements

  • WebSocket-enabled block stream for near-instant block propagation to the UI
  • Real-time transaction feed with live addition/removal of pending & confirmed transactions
  • Continuous contract deployment updates, ensuring new contract pages appear instantly
  • Optimized diff-based rendering to avoid full component reloads
     

Outcome

Users will see new blocks, transactions, and contracts the moment they are produced, creating an explorer experience on par with high-performance networks and modern monitoring tools.
 

Next Step

Internal event-bus testing → API-level subscription testing → Explorer UI integration.

Blockchain Updates

Relay Node Sync Validation & Peer Selection Upgrade

Testing is underway to resolve inconsistencies in relay node behavior, particularly cases where nodes attempted to relay through peers that were partially synced or lagging on the EVM layer.

The updated selection logic enforces a strict definition of “fully synced” to maintain network stability and prevent stale state propagation.
 

Definition of Fully Synced

A node must have BOTH:

  1. DAG Layer Fully Synced

2. EVM Layer Fully Synced

A node synced only on DAG but delayed on EVM will be automatically excluded.
 

Priority Model

Primary Selection:

  • Use a fully synced archive node as the relay target whenever possible.

Fallback Selection:

  • If no archive nodes qualify, select the best-performing non-archive peer that is fully synced across both layers.
     

Expected Impact

  • Eliminates relay misrouting
  • Reduces fork/lag scenarios in peer clusters
  • Improves block broadcast consistency and reduces delay variance
  • Enhances network uptime and overall sync reliability


Next

QA verification → load testing → propagation monitoring on staging env.

X1 & X10 Miner Updates

New Feature Design & Functional Planning

The product and design teams are currently in the discussion and exploration phase for the next feature set of the X1 Miner App. The focus at this stage is to evaluate which functionalities will bring the most value to users and how these features should visually and technically integrate into the existing app framework.
 

This includes early-stage concept shaping, UX direction checks, and reviewing multiple design outcome possibilities before finalizing the feature list. Parallel to this, the teams are also assessing UI refinement opportunities for the X10 Miner app to ensure consistency and improved clarity across devices.
 

The feature set is in the discussion phase, and the teams are actively iterating through design approaches, layout structures, and user-interaction flows to determine what will be included in the upcoming X1 release.

Open Items

  • WebSocket event schemas for Explorer streams
  • DAG & EVM sync verification unit tests for relay nodes
  • UX validation for upcoming X1/X10 feature sets
  • Inter-service sync testing for live data pipelines

Next Steps

  • Deploy WebSocket-based real-time updates for Explorer listings
  • Finalize relay node sync classification & apply logic cluster-wide
  • Complete X1/X10 feature definition → begin prototype phase
  • Conduct network-wide relay routing stability testing

Conclusion

From real-time visibility in the Explorer to stricter relay node validation and the next generation of miner-app features, this release continues to improve performance, transparency, and user experience across the BlockDAG ecosystem. With live WebSocket feeds approaching completion, relay node reliability hardening underway, and X1/X10 enhancements taking shape, BlockDAG is moving toward a faster, smarter, and more resilient decentralized infrastructure.
